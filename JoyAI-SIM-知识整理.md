# JoyAI-SIM 框架知识整理

## 一、核心定位

针对具身智能中**模型评测难以规模化**、**机器人训练数据难以低成本获取**两大问题，JoyAI-SIM 以仿真为中心枢纽，构建 **Robot ⇌ Simulation ⇌ Human** 双向工具链。工程实现依托京东云 JoyBuilder 平台，提供仿真执行、渲染、真实感增强和数据管理服务。

---

## 二、双向路径总览

```
Robot ──────────→ Simulation ──────────→ Human
（真机数据）        （仿真枢纽）         （人类数据）
  稀缺但贴近部署    可扩展、可诊断       丰富但不能直接执行

Human ──────────→ Simulation ──────────→ Robot
（人类示教/视频）    （物理验证）        （可部署轨迹）
```

| 路径 | 起点 | 目标 |
|------|------|------|
| Robot → Sim → Human | 真实机器人任务 | 人机对齐的模型评测 + 轨迹自然性检查 |
| Human → Sim → Robot | 人类示教视频 | 机器人可用的轨迹、标注与视觉观测 |

## 三、Human → Simulation → Robot（数据增强路径）

### 阶段 1：Human → Simulation

**输入**：第一人称视角的人体演示视频

**处理步骤**：
1. **手部动作恢复**：用 MediaPipe / HaMeR 等提取手部关键点和轨迹
2. **事件提取**：抓取/释放事件、物体交互关系
3. **场景重建**：提取场景几何信息，构建数字孪生环境

**输出**：具有空间结构、交互关系和任务语义的仿真实例

### 阶段 2：Simulation → Robot

**处理步骤**：
1. **轨迹重定向**：人手轨迹 → 机械臂末端 6DoF 轨迹（适配运动学约束）
2. **物理可行性过滤**：剔除关节限位违规、碰撞、不可达位姿、不合理接触
3. **仿真随机化（Domain Randomization）**：
   - 物体位置/组合变化
   - 容器布局变化
   - 颜色、材质变化
   - 光照、背景变化
4. **机器人视角渲染**：生成机器人观测视频

**输出**：多样化且物理可行的机器人轨迹 + 观察视频

## 四、Robot → Simulation → Human（评测与校准路径）

### 阶段 1：Robot → Simulation（数字孪生构建）

- 真实世界提供：场景布局 + 交互逻辑（物理规则）
- 仿真提供：可操作的虚拟 3D 资产

### 阶段 2：Simulation 内的合成数据生成

生成策略：
| 方法 | 说明 |
|------|------|
| Teleop（遥操作） | 人类键鼠控制虚拟机械臂 |
| IKFlow | 逆运动学生成可行路径 |
| FSM | 有限状态机，规则驱动 |
| RL | 强化学习自动探索 |

### 阶段 3：一致性校准（Consistency Calibration）

确保合成数据与真实世界物理一致：
- 动力学对齐
- 视觉对齐
- 行为对齐

校准完成后，人类可以在仿真中操作并产出可信数据。

## 五、数据约束的松弛层级

从最受限到最自由，每一步松开一个约束：

| 层级 | 数据形态 | 松开的约束 |
|------|----------|------------|
| 0 | Robot Video（真机视频） | — 固定机器人、场景、视角、外观 |
| 1 | Scene Alignment（场景重建） | 物理世界 → 搬入仿真 |
| 2 | Simulation Video（仿真视频） | 可在仿真中自由操作 |
| 3 | Action Generalization | 视角和轨迹 → 换视角、换路径 |
| 4 | Domain Randomization | 外观 → 换材质、换光照 |
| 5 | Human Video（人类视频） | Embodiment → 不同身体、不同场景 |

设计思路：**自顶向下** — 从少量真机数据出发，经仿真层层放大，最终利用互联网海量人类操作视频。

## 六、Human-Robot Aligned 模块

### 6.1 Model Evaluation（HR-AME）

**问题**：传统评测只看机器人在仿真中的成功率，缺乏参照基准。

**方法**：让人类在同一任务和仿真环境下执行相同任务，对比：
- 成功率 gap
- 效率 gap
- 路径效率 gap

### 6.2 Data Generation（HR-ADG）

**问题**：人类（5 指、第一人称、自然环境）与机器人（2 指夹爪、实验室环境）天然不对齐。

**方法**：在仿真中统一 — 人类操作员用键鼠控制虚拟机械臂，记录的直接就是机器人坐标下的 6DoF 轨迹 + 夹爪状态。

### 6.3 Human Embodied Feedback

**方法**：通过同步 VR 录制，操作员以具身视角检查机器人行为：
- Approach 策略是否符合直觉
- 接触时机
- 运动平滑度

过滤掉"物理可行但人类判断为别扭"的行为。

**四项评估指标**：

| 指标 | 含义 |
|------|------|
| Approach Check | 机械臂靠近目标的方向、角度、路径是否符合人类操作直觉 |
| Phase Transition | 各动作阶段切换是否自然（移动→接近→下降→抓取→抬起→搬运→放置→松开） |
| Contact Timing | 接触时机是否恰当 |
| Motion Smoothness | 运动是否平滑连贯 |

## 七、关键技术术语速查

| 术语 | 含义 |
|------|------|
| Digital Twin | 真实场景的仿真复现 |
| Domain Randomization | 随机化视觉外观（材质/光照/纹理），防止过拟合 |
| Retargeting | 将一种 embodiment 的动作映射到另一种 |
| Consistency Calibration | 仿真与真实世界的物理一致性校准 |
| 6DoF | 6 自由度（位置 xyz + 姿态 rpy） |
| IKFlow | 基于逆运动学的轨迹生成 |
| FSM | 有限状态机，规则驱动的行为生成 |
