# 首页粒子系统技术文档

## 一、系统概览

首页 3D 粒子效果基于 Three.js `Points` 实现。加载 `desk-split.glb` 桌面模型（65 个 mesh 部件），在模型表面均匀采样 50 万个粒子点。粒子初始状态为散布漂浮，用户点击后聚合还原为完整桌面模型形态。

**技术栈**：Three.js r0.160.0 + MeshSurfaceSampler + 自定义 ShaderMaterial + UnrealBloomPass 后处理

---

## 二、模型加载与预处理

### 2.1 模型来源

- 文件：`assets/desk-split.glb`
- 结构：包含 `tripo_part_0` ~ `tripo_part_64` 共 65 个独立 mesh
- 关键部件映射：
  - `tripo_part_5` → 桌面
  - `tripo_part_16 / 17 / 24 / 25` → 桌腿
  - 其余部件 → 桌上物体、机械臂等

### 2.2 自动缩放与定位

```
加载后流程：
1. 计算整体 bounding box
2. 取 maxDim = max(sizeX, sizeY, sizeZ)
3. 缩放比 autoScale = 6 / maxDim（归一化到 6 个单位）
4. Y 轴旋转 270°（3π/2），正面朝向用户
5. 位移补偿 = -center * autoScale（居中到原点）
```

### 2.3 相机参数

| 参数 | 值 |
|------|-----|
| FOV | 55° |
| 位置 | (-0.8, 0.6, 5.5) |
| 观察目标 | (-0.5, 0.4, 0) |

## 三、粒子采样

### 3.1 采样策略

使用 `MeshSurfaceSampler` 对每个 mesh 独立采样，按三角面积比例分配粒子数量，确保模型表面粒子密度均匀。

```
总粒子数 = 500,000
每个 mesh 分配数 = round((该 mesh 三角面数 / 总三角面数) × 500,000)
```

### 3.2 部件归属追踪

每个粒子记录所属 `partId`（从 mesh 名称 `tripo_part_X` 中提取编号），用于后续分区着色。

---

## 四、粒子初始散布

每个粒子的初始位置 = 模型表面采样点（home position）+ 随机球面偏移：

```
偏移半径 = 0.1 ~ 0.5（随机）
偏移角度 = 两个独立随机角 angle1, angle2 ∈ [0, 2π)

x_offset = cos(angle1) × radius
y_offset = sin(angle2) × radius × 0.3    ← Y 轴压缩，散布偏扁
z_offset = sin(angle1) × radius
```

散布后粒子紧贴模型表面形成薄雾状态，不会飘散太远。

### 漂浮动画（gathering 前）

粒子处于 `floating` 状态时持续做基于正弦的噪声漂移：

```
drift 幅度 = 0.15
每帧位移增量：
  dx = sin(time × 0.3 + i × 0.01) × 0.15 × 0.02
  dy = cos(time × 0.2 + i × 0.013) × 0.15 × 0.3 × 0.02
  dz = sin(time × 0.25 + i × 0.017) × 0.15 × 0.02
```

Y 方向漂移量只有 X/Z 的 30%，保持水平散布形态。

## 五、点击聚合动画

### 5.1 触发方式

- 点击 "CLICK TO CONVERGE" 引导按钮，或直接点击 canvas 背景
- 调用 `window.startGather()`，状态从 `floating` 切换为 `gathering`

### 5.2 聚合过程

```
总时长 = 6 秒
progress = min(elapsed / 6.0, 1.0)
ease = smoothstep(progress) = p² × (3 - 2p)    ← 先慢后快

每帧 lerp 速度 = 0.003 + ease × 0.077
  - 开始时：~0.003（缓慢启动）
  - 结束时：~0.08（快速收敛）

粒子位置更新：
  pos += (target - pos) × lerpSpeed
```

### 5.3 鼠标涡旋交互

聚合进度超过 80% 后启用。鼠标通过 Raycaster 投射到 Y=0 的水平面上获取 3D 坐标。

```
影响半径 = 2.2 单位
衰减 = (1 - distance / 2.2)²
强度 = falloff² × 0.036

偏移方式（涡旋而非推开）：
  target_x += (-dz / dist) × strength    ← 垂直于鼠标方向
  target_z += (dx / dist) × strength
  target_y += sin(time × 3 + dist × 6) × strength × 0.5  ← 上下波动
```

## 六、粒子着色

基于 **部件归属（partId）** + **空间位置** 的分区着色方案。先计算每个 mesh 的 bounding box，将粒子位置归一化到 [0, 1]。

### 6.1 桌腿（partId = 16/17/24/25）

深冷色调，几乎纯黑，带微弱蓝色：

```
R = 0.04 + nz × 0.02
G = 0.03 + nz × 0.02
B = 0.08 + nz × 0.04
```

### 6.2 桌面（partId = 5）

暗色底调带暖琥珀色光斑：

```
基础色：深棕暖色（R 0.1~0.18, G 0.06~0.1, B 0.08~0.12）
随机 6% 的粒子叠加琥珀高光：R +0.35, G +0.18, B +0.03
```

### 6.3 桌上物体与机械臂

按高度 `ny` 区分：

**机械臂（ny > 约 0.7）**：冷蓝-紫色调
```
R = 0.2 + heightBoost × 0.15
G = 0.15 + heightBoost × 0.15
B = 0.65 + heightBoost × 0.3

顶端 15% 概率出现蓝白高光：(0.75, 0.85, 1.0)
```

**桌上物体（ny < 0.7）**：三区域纯色渐变
```
左侧（nx → 0）：电光蓝 (0.08, 0.35, 0.95)
中间（nx → 0.5）：深靛紫 (0.2, 0.08, 0.6)
右侧（nx → 1）：热琥珀 (0.95, 0.45, 0.05)

权重使用 pow 幂函数实现区域分离，最终加权混合
```

### 6.4 全局色彩修正

```
深度暖化：前方粒子 R +nz×0.06, B -nz×0.04
噪声抖动：每粒子 ±0.12 随机偏移（R 全量、G ×0.3、B ×0.5）
亮点强化：ny > 0.6 的非桌面/桌腿粒子，3% 概率叠加 0.3~0.7 的高光
```

## 七、粒子大小

```
基础大小 = 0.4 + random() × 2.6          → 范围 [0.4, 3.0]
8% 的粒子额外 +2.0                       → 最大可达 5.0

Shader 中：
  屏幕大小 = size × breathe × (90 / -mvZ)
  clamp 到 [0.5, 6.0] 像素

breathe = 1 + sin(time × 0.1 + phase) × 0.02  ← 微弱呼吸缩放
```

## 八、Shader 实现

### 顶点着色器

- 读取自定义属性：`customColor`、`size`、`phase`
- 呼吸动画：基于时间和相位的正弦波微缩放
- 透明度固定 0.6 × breathe
- 点大小随深度缩放（透视近大远小）

### 片元着色器

- 圆形裁剪：距中心 > 0.5 的像素 discard
- 柔边：`smoothstep(0.5, 0.1, distance)` 从边缘到中心渐变

### 混合与渲染

| 参数 | 值 |
|------|-----|
| blending | NormalBlending |
| transparent | true |
| depthWrite | true |
| depthTest | true |

## 十、性能要点

- 粒子总数 50 万，使用单个 `Points` 对象 + `BufferGeometry`，避免逐粒子 draw call
- 像素比限制 `Math.min(devicePixelRatio, 2)`
- 帧间隔上限 `Math.min(dt, 0.05)` 防止切后台后的跳帧
- 抗锯齿关闭（`antialias: false`），依赖 bloom 柔化边缘
