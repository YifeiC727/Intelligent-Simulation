# JoyAI-SIM 交互体验 Storyline

> 核心原则：不管谁来，第一件事是让他**亲手操控机械臂**。解释和数据展示都放在体验之后。

---

## 第一幕：进入与好奇（30 秒内抓住用户）

**用户心理**：我听说这里可以操控机器人？

**页面**：落地页 / 具身交互入口

- 一段 3-5 秒的短视频/动图：机械臂在仿真场景中流畅地抓取物体
- 一句话 hook："你能比机器人做得更好吗？"
- 一个按钮：**「立即挑战」**

> 不要在这一步解释技术原理、平台定位、数据闭环。只制造好奇 + 挑战欲。

---

## 第二幕：极速上手（60 秒内完成第一次操作）

**用户心理**：让我试试

**页面**：操作引导 + 首个任务

### 2.1 场景加载（< 3 秒）

- 预设场景：书桌 / 茶几（简单、日常、无认知负担）
- 桌面上 3-5 个物体，目标位置有高亮提示

### 2.2 30 秒新手引导（边做边学，不要弹窗教程）

引导拆成 3 步，每步即时反馈：

| 步骤 | 操作 | 反馈 |
|------|------|------|
| ① 移动 | "拖动鼠标，把机械臂移到杯子上方" | 机械臂跟随，靠近时杯子高亮 |
| ② 抓取 | "按空格键，夹住杯子" | 夹爪闭合动画 + 触觉音效 |
| ③ 放置 | "移到目标区域，松开空格" | 放置成功 → 粒子特效 + 得分弹出 |

> 关键：第一次成功抓取必须在 30 秒内发生。如果用户卡住，给予辅助提示（箭头指引、放大目标区域等）。

### 2.3 首次完成

- "你完成了第一次分拣！用时 XX 秒"
- 自然过渡到正式任务："来试试完整挑战？"

---

## 第三幕：正式挑战（核心体验 3-5 分钟）

**用户心理**：我能做得更好 / 我想把桌面全整理完

**页面**：正式任务界面

### 3.1 任务内容

- 书桌上散落 8-12 个物体
- 目标：按类别放到指定区域（杯子→杯架、书→书架、笔→笔筒）
- 计时 + 实时得分

### 3.2 评分维度（即时显示，激发竞争心）

| 维度 | 用户看到的表述 | 背后对应 |
|------|--------------|---------|
| 速度 | "用时 2:34" | 任务效率 |
| 精准度 | "放置精度 87%" | 路径效率 |
| 流畅度 | "动作流畅度 A" | Motion Smoothness |
| 成功率 | "9/10 物体完成" | 端到端完成率 |

### 3.3 失败与重试

- 抓取失败 / 物体掉落：不惩罚，只是不得分，鼓励重试
- 任务超时：显示已完成部分的得分，"再来一次？"

---

## 第四幕：人 vs 机器（制造震撼感 / 思考）

**用户心理**：我做得怎么样？

**页面**：结果对比页

用户完成任务后，**不是只看自己的分数**，而是看到：

```
┌─────────────────────────────────────────┐
│          你的表现 vs 机器人表现            │
├──────────────┬──────────────────────────┤
│   你（人类）  │      机器人（AI）         │
│  ─────────── │  ───────────────────     │
│  用时 2:34   │  用时 3:12               │
│  精准度 87%  │  精准度 72%              │
│  流畅度 A    │  流畅度 B-               │
│  成功 9/10   │  成功 7/10              │
├──────────────┴──────────────────────────┤
│  ★ 你在精准度上超越了机器人 15%！          │
│                                         │
│  [ 查看机器人操作回放 ]  [ 再来一局 ]      │
└─────────────────────────────────────────┘
```

### 4.1 对比回放（关键展示环节）

- 左右分屏 / 叠加播放：用户的操作轨迹 vs 机器人的操作轨迹
- 高亮差异点：机器人在哪个阶段做得差（接近角度别扭、放置不精准等）
- 对应 HR-AME 的四个指标（但用用户能理解的语言呈现）

### 4.2 对老板 / PR 的价值

这个页面直接就是宣传素材：
- "普通用户 3 分钟上手就能超越当前 AI 模型"→ 说明数据采集的价值
- "AI 在 XX 方面还差 YY%"→ 说明技术提升空间和方向
- "已有 N 位用户贡献了 M 条操作轨迹"→ 说明数据飞轮在转

---

## 第五幕：留存与竞技（长期驱动）

**用户心理**：我想做到更好 / 想超过别人

**页面**：排行榜 + 进阶挑战

### 5.1 排行榜

- 综合得分排行
- 单项排行（最快、最精准、最流畅）
- 好友对比 / 全服排名

### 5.2 进阶难度

| 难度 | 变化 |
|------|------|
| 初级 | 3-5 物体，大目标区域 |
| 中级 | 8-12 物体，精确放置 |
| 高级 | 物体密集、遮挡、需要规划顺序 |
| 挑战 | 限时极速、单手操作、镜像模式等变体 |

### 5.3 对战模式（后续迭代）

- 实时双人竞速：同一场景谁先整理完
- 异步挑战：挑战好友的记录

---

## 第六幕：数据闭环展示（对老板 / 技术向受众）

**用户心理**（老板/合作方）：所以这有什么用？

**页面**：数据看板（可选入口，不强制所有用户看）

展示内容：
- 累计轨迹数据量（实时增长动画）
- 人类平均表现 vs 机器人表现的趋势图（机器人在逐步追上）
- 数据回流链路图：用户操作 → 轨迹采集 → 模型训练 → 真机部署
- 真机执行成功的对比视频（仿真 vs 真机）

> 这个页面就是向老板汇报时一键打开的 dashboard。

---

## Storyline 全景图

```
好奇进入 → 30s 上手 → 正式挑战 → 人机对比（震撼点）→ 排行竞技 → 数据闭环展示
   │            │            │              │                │              │
   ▼            ▼            ▼              ▼                ▼              ▼
 落地页      引导任务      核心游戏      结果对比页        排行/进阶      数据看板
(所有人)    (所有人)      (所有人)    (所有人+PR素材)   (C端留存)    (老板/合作方)
```

---

## 关键设计原则

1. **先体验，后解释** — 任何人进来不需要先读任何文档/介绍，直接动手
2. **30 秒内第一次成功** — 如果用户 30 秒内没有成功抓取一次物体，就会流失
3. **人机对比是核心叙事** — 不是"看机器人多厉害"，而是"看你和机器人的差距"，这个叙事同时满足 PR 传播 + 技术价值展示
4. **每一次用户操作都是数据** — 但用户不需要知道这件事，他只是在玩
5. **老板和 C 端看到的是同一个产品** — 不做两套，老板在 demo 时就是打开正式产品走一遍
