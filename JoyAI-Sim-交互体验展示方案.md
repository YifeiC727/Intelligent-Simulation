# JoyAI-Sim 交互体验平台方案

> 版本：v2.0 | 日期：2026-06-30

---

## 一、需求背景

JoyAI-Sim 具身智能团队已具备仿真环境搭建、机械臂轨迹生成与模型训练的核心能力，需要上线一个面向外部的**交互体验平台**，承载两个核心诉求：

| 诉求 | 说明 |
|------|------|
| **对外展示（PR）** | 向老板、合作方、博主及 C 端用户展示具身智能技术能力，提升品牌影响力 |
| **数据采集（技术演进）** | 通过用户操作大规模采集轨迹数据，替代现有 6DoF 鼠标采集方案，反哺真机抓取模型训练 |

> 两个目标相互促进：体验做得越好、用户越多，采集到的轨迹数据越丰富；模型效果越好，展示效果也越有说服力。

---

## 二、产品目标

提供一个**成熟的、浏览器可访问的交互体验平台**，面向所有受众（老板汇报、博主传播、合作方体验、C 端用户）提供统一入口、统一体验——所有人进来都是玩同一个游戏，不做多套 Demo。

核心目标：
1. **可体验**：任何人打开浏览器即可通过游戏化操作上手操控机械臂完成分拣任务
2. **可传播**：操作完成后的人机对比页面直接可作为 PR 传播素材，博主可自行录屏
3. **可增长**：游戏化设计驱动 C 端用户持续使用，每次操作静默贡献轨迹数据

### 受众与关注点

所有受众入口统一、体验统一（都是玩游戏），但各自关注的侧重点不同：

| 受众 | 核心关注点 | 平台需要满足的 |
|------|-----------|---------------|
| 老板 / 内部汇报 | 技术能力 + 商业价值 | 人机对比数据、数据回流闭环展示、一键可演示 |
| 外部合作方 / 博主 | 观赏性 + 传播性 | 逼真场景渲染、流畅操作录屏、人机对比结果页可截图分享 |
| C 端用户 | 上手门槛低 + 好玩 | 首次成功放置在 180 秒内、闯关递进、排行榜竞技 |

### 回流数据明细

用户每一次操作都会产出可用于模型训练的数据：

| 数据类型 | 具体内容 | 用途 |
|---|---|---|
| 操作轨迹 | 机械臂 6DoF 位姿序列（位置 xyz + 旋转 rpy），逐帧记录 | 训练抓取策略模型 |
| 夹爪状态 | 每帧开/闭状态 + 切换时间戳 | 学习抓取时机与力度控制 |
| 任务结果 | 成功/失败 + 失败原因（抓不住/掉落/放不准） | 构建奖励信号，用于强化学习 |
| 场景上下文 | 场景 ID、物体种类与初始位姿、目标位置 | 多场景泛化训练 |
| 用户操作路径 | 完整操作序列 + 路径选择 | 多样化策略数据，提升模型鲁棒性 |
| 操作元数据 | 任务耗时、重试次数、操作帧率 | 数据质量筛选与清洗 |

## 三、设计挑战与解决思路

| 挑战 | 问题描述 | 解决思路 |
|------|---------|---------|
| **技术挑战：操作难度极高** | 现有轨迹采集依赖专业 6DoF 鼠标，专业人员平均 30 分钟才成功一次抓取。普通用户只有 2DoF 普通鼠标 + 键盘组合键，操控映射违反直觉，30 秒内无法成功则直接流失 | **分段控制**：粗定位（系统辅助）→ 精细操作（用户手动，数据价值最高）→ 搬运（用户指定目标，规划器执行路径）。支持慢动作模式 + 放大视角，操控不需要快，只需要准 |
| **留存挑战：用户激励不足** | "操控机械臂"对普通用户无天然吸引力——非刚需、无社交属性、缺乏即时反馈和成就感。没有激励机制则用户试一次就走 | **游戏化竞技**：以"你能比机器人做得更好吗？"为核心 hook。人机对比得分、闯关递进（初级→高级）、排行榜（综合 + 单项）、多路径探索天然产出多样化轨迹数据 |

## 四、用户旅程（Storyline）

> 核心原则：**先体验，后解释**。任何人进来不需要先读文档/介绍，直接动手。

### 全景图

```
好奇进入 → 30s 上手（首次人机对比）→ 正式挑战（实时人机对比）→ 排行竞技
   │            │                          │                          │
   ▼            ▼                          ▼                          ▼
 落地页      引导任务                    核心游戏                    排行/进阶
(所有人)    (所有人)                    (所有人)                   (C端留存)
```

---

### 第一幕：进入与好奇（30 秒内抓住用户）

**用户心理**：我听说这里可以操控机器人？

**页面**：落地页 / 具身交互入口

- 一段 3-5 秒的短视频/动图：机械臂在仿真场景中流畅地抓取物体
- 一句话 hook："你能比机器人做得更好吗？"
- 一个按钮：**「立即挑战」**

> 不要在这一步解释技术原理、平台定位、数据闭环。只制造好奇 + 挑战欲。

---

### 第二幕：极速上手（180 秒内完成首次放置任务）

**用户心理**：让我试试

**页面**：首个任务（引导融入操作）

**场景加载（约 60 秒）**：预设收纳场景，桌面上 3 个物体，目标位置有高亮提示。加载期间展示引导内容，确保等待时间不被浪费：

| 加载阶段 | 展示内容 |
|---------|---------|
| 0-20s | 场景预览图 + "即将进入：书桌整理挑战" |
| 20-40s | 操控方式动图（鼠标移动 / 空格抓取） |
| 40-60s | "你能比机器人做得更好吗？" + 进度快完成的暗示 |

直接进入任务，前 1-2 个物体带逐步引导提示，之后提示渐隐（边做边学，不要弹窗教程）：

| 步骤 | 操作 | 反馈 |
|------|------|------|
| ① 移动 | "拖动鼠标，把机械臂移到杯子上方" | 机械臂跟随，靠近时杯子高亮 |
| ② 抓取 | "按空格键，夹住杯子" | 夹爪闭合动画 + 音效 |
| ③ 放置 | "移到目标区域，松开空格" | 放置成功 → 特效 + 得分弹出 |

关键设计约束：
- 首次成功放置应在 **180 秒内**完成（含引导辅助）
- 如果用户卡住，给予渐进式辅助（箭头指引 → 放大目标区域 → 操作辅助）
- 第 1 个物体放好后不打断，只说 **"漂亮！还有 2 个等着你"**，保持心流
- 第 3 个物体起引导提示消失，用户凭手感操作

**计时策略**：不显示倒计时，不制造压力。系统后台静默计时 8 分钟：

| 用户情况 | 触发什么 | 展示什么 |
|---------|---------|---------|
| 5 分钟内全部完成 | 直接进结果页 | "你用了 XX 秒完成 3 个，机器人用了 YY 秒" → 时间对比 |
| 5 分钟到了未全部完成 | 角落出现跳过按钮 | "同样时间里，机器人完成了 3 个——要继续还是看结果？" |
| 用户点跳过 | 进入结果页 | 展示已完成部分 vs 机器人对比 |
| 用户忽略跳过，继续做 | 不打断，按钮保留 | 做完后出对比页，但时间劣势更大 |

**结果页根据表现展示不同钩子**：

| 结果 | 钩子文案 |
|------|---------|
| 比机器人快很多（>20%） | "你比机器人快了 XX 秒——物体多了之后还能赢吗？" |
| 和机器人差不多（±20%） | "和机器人不相上下——进阶赛中一决高下？" |
| 比机器人慢 | "机器人这次赢了——再试一次还是直接挑战进阶？" |

自然过渡到进阶挑战。

---

### 第三幕：进阶挑战（核心玩法 10-30 分钟）

**用户心理**：我能做得更好 / 我想把桌面全整理完

**页面**：正式任务界面

**任务内容**：书桌上散落 8-12 个物体，按类别放到指定区域（杯子→杯架、书→书架、笔→笔筒），**15 分钟倒计时**，实时得分。

**数据多样性与游戏设计对应关系**：

| 数据类型 | 例子 | 训练价值 | 游戏设计方法 |
|---------|------|---------|-------------|
| 同场景 · 同物体 · 不同抓取轨迹 | 都是先拿杯子，但每个人靠近角度、抓取姿态不同 | 最高——同一个任务的多种执行方式，直接用于策略模型泛化 | 固定关卡，鼓励重复玩同一关，每次操作自然有差异 |
| 同场景 · 同物体 · 不同顺序 | A先拿杯子再拿笔，B先拿笔再拿杯子 | 高——学习任务规划和排序决策 | 固定关卡 + 分数策略设计（不同顺序不同得分），引导顺序分化 |
| 不同场景 · 不同物体摆放 | 杯子这次在左边，下次在右边 | 高——场景泛化能力，但需要大量场景 | 每局随机化物体位置，用户每次面对新布局 |
| 不同场景 · 不同物体种类 | 这局是杯子+书，下局是瓶子+盒子 | 中——物体种类泛化，但每种物体的样本量会被稀释 | 物体池大、每局随机抽取不同组合 |

**物体差异化得分**：不同物体根据操作难度分配不同分数，场景开始时展示每个物体的分值：

| 物体特征 | 分数 | 设计意图 |
|---------|------|---------|
| 大物体、好抓（杯子、书） | 低分（10-20） | 简单操作，数据价值低 |
| 小物体、难抓（笔、回形针） | 高分（30-50） | 精细操作，数据价值高 |
| 需要特定朝向放置的 | 额外加分 | 旋转操作数据更有价值 |
| 连续成功不掉落 | 连击加分（x2, x3...） | 鼓励认真操作，提高成功轨迹占比 |

策略分化效果：稳健派先拿简单物体保底，冲分派直奔高分难物体，连击派追求不失误的连续操作 → 天然产出多样化轨迹数据。

**实时人机对比**：界面一侧展示机器人操作同一任务的画面（预录视频或机器人实时操作），让用户直观看到机器人的操作方式和进度。画面上方叠加关键数据对比：

- **完成数量**（主指标，大字）："你 3/8 · 机器人 4/8" — 一眼能看懂，直接制造紧迫感
- **总分**（辅助指标，小字）："你 180 分 · 机器人 150 分" — 体现策略差异（数量落后但分数领先 = 难物体拿得多）

**评分维度**（即时显示，激发竞争心）：

| 维度 | 用户看到的表述 | 背后对应的技术指标 |
|------|--------------|-----------------|
| 总分 | "280 分"（机器人 210 分） | 物体难度加权 + 连击倍率的综合得分，反映整体策略和执行水平 |
| 完成数 | "完成 9/12"（机器人 7/12） | 端到端成功抓取并放置的物体数量，最直观的能力指标 |
| 精准度 | "放置精度 87%"（机器人 72%） | 物体最终放置位置与目标位置的偏差，偏差越小精准度越高 |
| 流畅度 | "动作流畅度 A"（机器人 B-） | Motion Smoothness：机械臂运动路径是否平滑连贯，急停、抖动、反复折返会降低评分 |
| 最佳连击 | "x5 连击"（机器人 x3） | 连续成功放置物体且中间无掉落失误的最长记录 |
| 路径创意度 | "路径创意度 ★★★"（机器人 ★★） | 有待商榷。Approach Check：靠近物体的方向、角度是否与主流路径有差异且仍然合理，衡量轨迹多样性 |

**任务完成/时间到**：倒计时结束直接结算已完成部分（不是"失败"），展示汇总对比结果页。

结果页核心内容——人机对比数据面板：

| 维度 | 你 | 机器人 |
|------|---|-------|
| 总分 | 280 分 | 210 分 |
| 完成数 | 9/12 | 7/12 |
| 精准度 | 87% | 72% |
| 流畅度 | A | B- |
| 最佳连击 | x5 | x3 |
| 路径创意度 | ★★★ | ★★ |

设计处理：领先方数字高亮（如加粗 + 主题色），落后方灰显，不需要额外文字标注胜负。底部汇总一句话："你在 X/6 项中超越了机器人"。

这个页面同时是 PR 传播素材，底部附带传播话术（根据对比结果动态生成）：
- "普通用户 3 分钟上手就能超越 AI" → 数据采集的价值
- "AI 在 XX 方面还差 YY%" → 技术提升方向
- "已有 N 位用户贡献了 M 条轨迹" → 数据飞轮在转

**对比回放**（可选查看）：
- 左右分屏播放：用户的操作轨迹 vs 机器人的操作轨迹
- 高亮差异点：机器人在哪个阶段做得差

**失败与重试**：抓取失败 / 物体掉落扣时间（-3 秒），给用户"小心点"的动力但不过度惩罚。15 分钟内全清 12 个物体 → 额外时间奖励分。

### 第四幕：留存与竞技（长期驱动）

**用户心理**：我想做到更好 / 想超过别人

**页面**：排行榜 + 进阶挑战

**排行榜**：综合得分排行 + 单项排行（最快、最精准、最流畅）+ 好友对比 / 全服排名

**进阶难度**：

| 难度 | 变化 |
|------|------|
| 初级 | 3 物体，大目标区域 |
| 中级 | 8-12 物体，精确放置 |
| 高级 | 物体密集、遮挡、需要规划顺序 |
| 挑战 | 限时极速、单手操作、镜像模式等变体 |

**对战模式**（后续迭代）：实时双人竞速 / 异步挑战好友记录

**UGC 场景创建**（后续迭代）：
- **拍照建模**：用户拍摄自己桌面照片，系统生成对应的 3D 收纳场景
- **自由搭建**：用户从网站已有的 3D 素材库中拖拽物体，自行设计关卡布局
- **分享挑战**：用户创建的场景可发布为公开关卡，邀请其他人来挑战（"来整理我的桌面"）
- 传播价值：UGC 内容天然适合社交分享，同时每个新场景都扩展了训练数据的场景覆盖度

---

### 设计原则

1. **先体验，后解释** — 任何人进来直接动手，不需要先读任何介绍
2. **30 秒内第一次成功** — 如果用户 30 秒内没有成功抓取一次物体，就会流失
3. **人机对比是核心叙事** — 不是"看机器人多厉害"，而是"看你和机器人的差距"
4. **每一次用户操作都是数据** — 但用户不需要知道这件事，他只是在玩
5. **所有人看到的是同一个产品** — 不做两套，老板 demo 时就是打开正式产品走一遍

## 五、功能范围与优先级

### P0：MVP（单场景可操作 + 轨迹采集）

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 场景 | 书桌整理场景 | 桌面 3 个物体，目标位置有高亮提示 |
| 操控 | 键鼠控制机械臂 | 鼠标控制位姿移动，键盘控制夹爪开闭 |
| 引导 | 30 秒新手引导 | 边做边学：移动→抓取→放置，每步即时反馈 |
| 判定 | 成功/失败判定 | 抓取失败、掉落、放置失败三种情况 |
| 数据 | 轨迹静默采集 | 6DoF 位姿 + 夹爪状态 + 任务结果，后端存储 |
| 渲染 | three.js + WebGL2 | 资产预加载，确保流畅 |

### P1：人机对比 + 闯关

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 对比 | 人 vs 机器人得分对比 | 用时、精准度、流畅度、成功率逐项对比 |
| 回放 | 操作轨迹回放 | 用户轨迹 vs 机器人轨迹分屏播放 |
| 闯关 | 多难度关卡 | 初级（3 物体）→ 高级（密集+遮挡） |
| 评分 | 多维评分系统 | 速度、精准度、流畅度、成功率 |

### P2：留存与传播

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 排行 | 排行榜 | 综合 + 单项排名 |
| 场景 | 预设多场景 | 茶几整理、厨房收纳等（基于 5w+ 资产库） |
| 场景 | 拍照生成 3D 场景 | 用户拍摄真实桌面照片，系统自动生成对应 3D 收纳场景 |
| 场景 | 自由搭建场景 | 用户从平台素材库中拖拽 3D 物体，自行设计关卡布局 |
| 社交 | UGC 场景分享 | 用户创建的场景可发布为公开关卡，邀请他人挑战 |
| 社交 | 对战模式（可选） | 双人实时竞速 |

### P3：数据闭环可视化

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 看板 | 数据回流仪表盘 | 累计轨迹量、人 vs 机器人趋势图 |
| 对比 | 真机 vs 仿真视频 | 展示数据回流到真机的效果 |
